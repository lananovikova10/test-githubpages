Switchable content

To add larger pieces of switchable content to the page, use switcher-key tag. This is an alternative to Tabs.

For example, if a feature's description is different for Windows and macOS, you need to provide it for each platform individually in the topic, or if in general documentation is the same, but you need to provide examples in different languages or frameworks.

Create switchable content

  1. Add the switcher-key attribute to the elements that should appear when the user selects a certain switcher on top of the page.

    We do not support using an if element with the switcher-key, because there is no way to wrap multiple elements and switch them in the output.

  2. (Optional) Add the switcher-label attribute to the topic tag if there is an entity that sections describe. For example, operating system, programming language, technology and so on.

Use the switcher at the top of the page to switch chapters in the example below:

Markup

Result

<chapter title="Section One" switcher-key="Section One"> Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. </chapter> <chapter title="Section Two" switcher-key="Section Two"> Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat. </chapter>
