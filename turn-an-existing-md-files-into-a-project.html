Tutorial: Turn existing MD files into a documentation project

From this tutorial, you will learn how to turn a bunch of existing Markdown files that you already have locally into a documentation project.

We will not only import but put them into a meaningful hierarchy and add a few custom bits into basic Markdown syntax.

Import .md files

  1. Copy your existing .md files into the project's /topics folder.

  2. A top-level heading (#) will be considered a topic's title. Do not use it more than once per topic.

  3. In the Writerside tool window, click or press Alt+Insert.

  4. Select Link Topic Files to TOC..., choose file you want to add to the tree.

  5. If you have images in your .md files, open the Project view and put them into the /images folder.

Change the topic's hierarchy

Now our newly added topics are placed flat, so let's put them into a hierarchy: make one of the pages a section and another one a wrapper for other pages.

To transform a topic into a section that groups a few topics under:

  1. Right-click the topic title and select Edit Toc Element.

  2. Remove / self-closing element and make a separate closing tag for this topic tree record.

  3. Place other topics that relate to this section inside this topic.

    <toc-element topic="Getting_started.md"/> <toc-element topic="Installation.md"/> <toc-element topic="Configuration.md"/>
    <toc-element topic="Getting_started.md"> <toc-element topic="Installation.md"/> <toc-element topic="Configuration.md"/> </toc-element>

To make a wrapper that groups a few topics under, but has no content in it:

  1. Go to the instance's .tree file.

  2. Create an element with a toc-title but without a topic attribute.

  3. Place other topics that relate to this section inside this element.

<toc-element toc-title="Getting started"> <toc-element topic="Installation.md"/> <toc-element topic="Configuration.md"/> </toc-element>

Enhance code blocks with semantic markup elements

We imported some pure Markdown topics, now we can make them more sophisticated using semantic markup. In this tutorial, we will touch up the code blocks.

  1. In the Writerside tool window, click the topic title in the tree and find the code block.

  2. To make the large code block collapsible, add the collapsible="true" attribute to the code block.

  3. By default it will be collapsed, to make it expand add default-state="collapsed" attribute.

  4. (Optional) By default, when a block is collapsed, the first line is displayed, to override it with another title, add collapsed-title="Title to display when a block is collapsed".

  5. (Optional) By default, the first line of the code snippet is used as a title.

    Change it by adding the collapsed-title attribute with a custom value or add the collapsed-title-line-number attribute to specify the line to use as the title.

``` ...your code block... ``` {collapsible="true" default-state="expanded" collapsed-title="Title to display when a block is collapsed"}

In this tutorial, we have imported some .md files, improved the hierarchy, and enhanced the code blocks.

On top of that, you can use any other semantic elements, like tabs, definition lists, or admonition blocks (tips, notes, and warnings).
