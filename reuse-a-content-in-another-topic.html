Reuse a content in another topic

To avoid content duplication in several places, you can reuse any part of the content in any number of topics. When you update the original content, Writerside will update the content in other places.

Warnings and other admonitions are often repeated and so are good candidates for content reuse.

Make a piece of content reusable

So, for example, you need to have the same warning on top of several topics, so you can have it written in a single place and reference it in all others.

  1. Create a separate topic for the warning.

    <topic title="Warnings" id="Warnings_library" is-library="true">

  2. Add the is-library="true" attribute to the topic element. Library topics are not included in the table of contents.

  3. Wrap a warning content in the snippet element with an id. Do not use whitespaces in the id.

    <snippet id="generic_warning"> <warning>This version support ended on January 14, 2020. Download the latest version.</warning> </snippet>

Do not use snippet id to reference the anchor, the link will be broken.

Include reusable content

You can reuse the snippet with the same ID multiple times, even in the same topic. It is also handy for table headings with the same set of columns.

  1. To include reused content, use the include element.

  2. Set the from attribute that points to the topic where the included content resides and the element-id attribute that matches the id attribute of the snippet.

    <include from="warning_topic.xml" element-id="generic_warning"/> <procedure title="Do some work"> <step> Do this. </step> <step> Do that. </step> </procedure>
