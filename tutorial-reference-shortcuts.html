Tutorial: Reference shortcuts from the configuration file

If your plugin relies on keyboard shortcuts, you can reference them in docs by IDs from the keymap.xml file so that you won't need to manually specify a shortcut for each platform – the user will be able to pick a desired one on top of the page.

  1. Place keymap.xml with the shortcuts for actions your plugin uses in the project root.

    <?xml version="1.0" encoding="UTF-8"?> <Keymap id="instance_id"> <Action id="$Uncomment"> <Description>Uncomment or remove column mapping</Description> <Shortcut layout="primary_Win/Linux">Ctrl+B</Shortcut> <Shortcut layout="primary_macOS">⌘+B</Shortcut> </Action> <Action id="$Generate"> <Description>Generate Menu</Description> <Shortcut layout="primary_Win/Linux">Alt+Insert</Shortcut> <Shortcut layout="primary_macOS">⌘+N</Shortcut> </Action> <Action id="$Configure"> <Description>Configure classes</Description> <Shortcut layout="primary_Win/Linux">Alt+Enter</Shortcut> <Shortcut layout="primary_macOS">⌥+⏎</Shortcut> </Action> </Keymap>

  2. Place the platforms.xml file in the /cfg directory.

    In the id attribute of the platform element specify primary or secondary.

    In the use-keymaps element specify which keymaps can user select from in the picker on top of the page. All values should start with either primary_ or secondary_ prefix. The prefix won't be displayed to a user.

    In the src tag specify the keymap file name.

    <?xml version="1.0" encoding="UTF-8"?> <platforms> <platform id="primary"> <src>keymap.xml</src> <use-keymaps> primary_Win-Linux </use-keymaps> <use-keymaps> primary_macOS </use-keymaps> </platform> </platforms>

  3. Now reference the shortcut by a key in the topic:

    <p>Press <shortcut key="$Configure"/> on any element that you need to configure, and select <control>Show plugin popup</control>.</p>
