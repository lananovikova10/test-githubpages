Definition lists

To create definition lists, for example, to specify terms and their definitions, use the following markup. Specify the term on the first line. On the next line, type a colon followed by a space and the definition.

It also accepts multiline definitions and lists, including nested ones.

Use style attribute to specify the list layout.

Markup

Result

{style="narrow"} First Term : This is the definition of the first term. Second Term : This is the definition of the second term.
First Term

This is the definition of the first term.

Second Term

This is the definition of the second term.

{style="wide"} /customers : Get customers. /customers/{customerId} : Get customer by ID.
/customers

Get customers.

/customers/{customerId}

Get customer by ID.

{style="medium"} getFoo() : Returns the current value of Foo. setBar(String) : Assigns the provided string to Bar.
getFoo()

Returns the current value of Foo.

setBar(String)

Assigns the provided string to Bar.

{style="compact"} -o : Open a file. -c : Close a file.
-o

Open a file.

-c

Close a file.

Currently, it is not possible to apply other attributes, like style or sorting, to the Markdown definition lists, so if you need them, use the semantic markup.
