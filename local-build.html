Build an instance locally

Now when you finished writing some content, let's make your first local build of the docs.

A local build is handy if you want to preview the help instance as a static HTML, distribute the files directly or deploy it.

It is important to test your build locally before deploying it for production. Writerside lets you build ZIP archives with HTML files and images locally.

Build locally

  1. Right-click the instance name in the Writerside tool window and select Generate website | Save as ZIP Archive.

  2. If needed, change the configuration name, by default it is Instance name (ZIP). A local build action will be available under this name.

  3. By default, the archive will be saved to the help directory. If you need to specify another one, in the Save to field select a directory you want to save the output.

  4. Click Run.

  5. This will build two archives: one with HTML files and one with images.

    Unpack them, rename the images folder to images and put it inside the folder with HTML files.

Now you can either open the HTML files locally to take a look at them or deploy them by copying the webHelp/ directory to your production web server.
