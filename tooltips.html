Tooltips

To explain the meaning of a term or provide additional information without wasting space, use a tooltip attribute.

It displays a text when a user hovers over it.

Add a tooltip

  1. In the cfg/glossary.xml file of your documentation project, add the term you want to explain.

    For example, to specify an explanation for the GUI term, add the following line:

    <term name="GUI">Graphical User Interface</term>

  2. Go back to your topic file and wrap the text you want to explain into the tooltip tag with the term attribute.

Markup

Result

<p> The documentation branch for the next product version is built and published with the <tooltip term="GUI"> GUI </tooltip> status. </p>
